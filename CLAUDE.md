# EazyPlans

멀티세션 작업에서 플랜이 사라지지 않도록 파일로 고정하는 시스템.

> 이 파일을 ~/.claude/CLAUDE.md에 병합하거나 프로젝트 .claude/CLAUDE.md에 추가해서 사용.

---

## 핵심 원칙

Claude Code Plan Mode의 결과는 컨텍스트에만 존재한다. 세션이 길어지면 압축되어 사라진다.
이 시스템은 Plan Mode 결과를 파일로 굳혀서 어느 세션에서든 다시 읽을 수 있게 한다.

---

## 폴더 구조

```
plans/
├── active/     ← 현재 진행 중
├── backlog/    ← 대기 중 (우선순위 낮음)
└── done/       ← 완료
```

팀 협업 시: `plans/` 폴더를 git에 커밋하면 팀 전체가 같은 플랜을 공유한다.

---

## 플랜 생성 규칙

멀티세션 작업(한 대화에서 끝나지 않는 작업) 감지 시 `plans/active/`에 폴더 생성:

```
plans/active/YYYYMMDD-작업명/
├── research.md    ← [필수] 선행 리서치
└── overview.md    ← 전체 진행 오버뷰 + 단계별 체크리스트
```

단일 세션으로 끝나는 작업은 플랜 파일 불필요.

---

## Plan Mode 연계

```
[1] Plan Mode 진입
    → 관련 파일 Read (기억 기반 플랜 금지)
    → WebSearch/Context7로 최신 정보 확인

[2] research.md 작성 (ExitPlanMode 전 필수)
    → 라이브러리 버전, 베스트 프랙티스, 기존 코드 현황
    → 예외: 외부 의존성 없는 내부 리팩토링은 생략 가능

[3] overview.md 작성
    → 단계별 체크리스트 + [보고] 마커

[4] GATE -1 실행 (ExitPlanMode 직전 자동)
    → 5항목 체크 후 Build 전환
```

---

## research.md 규칙

- overview.md 작성 **전에** 반드시 먼저 작성
- 포함 내용: 라이브러리 비교, 최신 버전, 기존 코드 충돌 여부, 베스트 프랙티스
- 기억이 아닌 도구(WebSearch, Context7)로 수집
- 예외: 외부 의존성 없는 내부 리팩토링은 생략 가능

---

## overview.md + [보고] 마커

단계별로 `[보고]` 마커를 붙이면 해당 단계 완료 후 결과를 보고하고 다음 진행 전에 대기한다.

```markdown
## 단계
1. 리서치 → [보고]     ← 완료 후 보고 + 승인 대기
2. 설계 → [보고]       ← 완료 후 보고 + 승인 대기
3. 구현                ← 보고 없이 자율 진행
4. 검증 → [보고]       ← 완료 후 보고 + 승인 대기
```

- `[보고]` 있는 단계: 완료 → 보고 → 사용자 승인 후 다음 단계
- `[보고]` 없는 단계: 자율 진행
- 전문 판단이 필요한 단계(설계 방향, 비즈니스 로직)에 붙이는 것을 권장

---

## GATE -1 (Plan → Build 전환 검증)

ExitPlanMode 직전 자동 실행. 5항목 중 3개 미만이면 플랜 보완 요청.

```
[ ] 근본 원인이 명확한가?
[ ] 영향받는 파일을 식별했는가?
[ ] 완료 기준이 구체적인가?
[ ] 기존 코드와 충돌을 확인했는가?
[ ] 검증 방법이 있는가?

결과: N/5
→ 3개 미만: 플랜 보완 필요
→ 3개 이상: Build 전환 가능
```

---

## 라이프사이클

```
작업 시작  → plans/active/에 생성
보류       → plans/backlog/으로 이동
완료       → plans/done/으로 이동
```

---

## 팀 협업

```bash
# plans/ 폴더를 git에 포함
git add plans/
git commit -m "plan: YYYYMMDD-작업명 추가"
```

- 팀원이 같은 플랜 파일을 읽고 작업 맥락을 공유
- PR 시 overview.md가 변경 이유의 근거 문서로 활용
- done/으로 이동한 플랜은 의사결정 히스토리로 보존

---

## 기존 시스템과 역할 분리

| | EazyPlans | Plan Mode |
|--|-----------|-----------|
| 범위 | 멀티세션 (파일로 영속) | 단일 세션 내 |
| 산출물 | research.md + overview.md | Task (컨텍스트) |
| 소멸 여부 | 사라지지 않음 | 압축 시 사라짐 |
| 팀 공유 | git 기반 | 불가 |
